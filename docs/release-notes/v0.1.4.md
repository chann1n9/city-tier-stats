# v0.1.4

## 重点变化

- 新增按另一列分组统计城市分层的能力。程序仍会先输出原有的整体统计，再追加各分组内部的数量和占比。
- Windows 右键菜单、Windows“发送到”菜单和 macOS Finder Quick Action 现在默认按“企微名称”列追加分组统计。

## 新增

- 新增 `--group-by` 参数；单独使用时默认按“企微名称”列分组，也可以通过 `--group-by 列名` 指定其他列。
- 支持同时读取归属地列和分组列，并保持同一行数据的对应关系。
- 空白分组统一显示为“（空白）”，分组按输入文件中首次出现的顺序输出。

## 修复

- 修正 Windows“发送到”菜单名称中 `Multiple` 的拼写。

## 变更

- Windows 和 macOS 安装器创建的快捷入口会自动追加 `--group-by`；直接从命令行运行时仍可自行决定是否启用分组统计。
- 项目版本从 `0.1.3` 升级为 `0.1.4`。

## 安装包

- Windows: `city-tier-stats-0.1.4-setup.exe`
- macOS: `city-tier-stats-0.1.4.pkg`

## 已知问题

- 通过 Windows 或 macOS 安装器创建的快捷入口分析文件时，输入文件需要包含“企微名称”列；没有该列时会提示缺少列。可以改用命令行且不传 `--group-by` 来只输出整体统计。
